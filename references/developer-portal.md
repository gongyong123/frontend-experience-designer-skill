# Developer Portal

Goal: help developers understand, trust, and integrate quickly.

## Visual Rules

- Prioritize readable docs, strong navigation, and excellent code examples.
- Use clear hierarchy: overview, quickstart, concepts, API reference, examples.
- Keep code blocks legible and copyable.
- Show status, versions, and prerequisites clearly.

## Motion Rules

- Motion intensity should be low to medium.
- Use subtle navigation feedback, code tab transitions, copy confirmation, section reveal.
- Use product-like motion only in overview or landing sections.

## Good Patterns

- Sticky side navigation.
- Search and command palette.
- Copy button with clear success feedback.
- Language tabs.
- Endpoint cards with method colors.
- Quickstart progress steps.

## Avoid

- Heavy hero motion inside documentation pages.
- Animations that move code while reading.
- Hidden navigation.
- Decorative backgrounds behind dense text.
